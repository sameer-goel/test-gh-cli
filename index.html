Context Engineering for AI Agents

Lessons from building Manus. Six principles that shape how production agents think, remember, and recover.

Based on Yichao 'Peak' Ji's blog post from Manus

↓
Principle 01
🔑

Design Around the KV-Cache

The KV-cache hit rate is the single most important metric for a production agent. It directly affects both latency and cost. In Manus, the average input-to-output token ratio is around 100:1, making cached prefills critical.

Key practices Keep your prompt prefix stable (no timestamps at the start). Make context append-only. Mark cache breakpoints explicitly. Even a single-token difference invalidates the cache from that point onward.
Principle 02
🎭

Mask, Don't Remove

As your agent gains more tools, the action space explodes. Dynamically adding or removing tools mid-iteration breaks the KV-cache and confuses the model when previous actions reference removed tools.

Manus approach Use a context-aware state machine to mask token logits during decoding. Tools stay defined but are selectively enabled. Tool names share prefixes (browser_*, shell_*) for easy group masking.
Principle 03
📁

Use the File System as Context

Even 128K token windows aren't enough for real agentic work. Long inputs degrade performance and cost a fortune. The file system is unlimited, persistent, and directly operable by the agent itself.

Design principle Compression strategies must be restorable. A URL can replace a full web page. A file path can replace a document's contents. The agent writes and reads on demand, using the file system as externalized memory.
Principle 04
📋

Manipulate Attention Through Recitation

Manus creates a todo.md and updates it step-by-step. This isn't just organization. By rewriting the plan at the end of the context, the agent pushes its goals into the model's recent attention span.

Why it works A typical Manus task requires ~50 tool calls. Without recitation, the model drifts off-topic in long contexts. The todo list acts as a natural language attention bias, avoiding "lost-in-the-middle" problems.
Principle 05
❌

Keep the Wrong Stuff In

Agents make mistakes. The impulse is to hide errors, clean the trace, retry silently. But erasing failure removes evidence. Without evidence, the model can't adapt or avoid repeating the same mistake.

The insight When the model sees a failed action and the resulting error, it implicitly updates its beliefs. This shifts its prior away from similar actions. Error recovery is one of the clearest indicators of true agentic behavior.
Principle 06
🔀

Don't Get Few-Shotted

LLMs are excellent mimics. If your context is full of similar action-observation pairs, the model follows that pattern even when it's no longer optimal. Repetition breeds drift and hallucination.

The fix Introduce structured variation: different serialization templates, alternate phrasing, minor noise in formatting. The more uniform your context, the more brittle your agent becomes. Controlled randomness breaks the pattern.
🎯

The Agentic Future Will Be Built One Context at a Time

Models may be getting stronger, faster, and cheaper, but no amount of raw capability replaces the need for memory, environment, and feedback. How you shape the context ultimately defines how your agent behaves: how fast it runs, how well it recovers, and how far it scales.
